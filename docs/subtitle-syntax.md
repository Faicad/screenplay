# 字幕语法

`.mjs` 脚本中的 `const subtitle = \`...\`` 支持以下语法控制字幕显示和 TTS 朗读行为。

## 1. 语音控制 `{voiceName}`

放在行首，指定该行字幕使用的 edge-tts 语音。

```
{zh-CN-YunxiaNeural}((提示词：))帮我改成田园风格的环境贴图
```

| 项目 | 结果 |
|------|------|
| 字幕显示 | `提示词：帮我改成田园风格的环境贴图` |
| TTS 朗读 | `帮我改成田园风格的环境贴图`（`zh-CN-YunxiaNeural`） |
| 缺省语音 | `zh-CN-XiaoxiaoNeural` |

`{voiceName}` 本身不会出现在字幕显示和 TTS 文本中。

## 2. 括号语法

### `(text)` 或 `（text）`

括号及其内容**不朗读**，但在字幕中**显示**（含括号）。

```
安装也很简单，以work buddy为例\n (也支持claude code)
```

| 项目 | 结果 |
|------|------|
| 字幕显示 | `安装也很简单，以work buddy为例\n (也支持claude code)` |
| TTS 朗读 | `安装也很简单，以work buddy为例` |

### `((text))` 或 `（（text））`

双层括号内容**不朗读**，但在字幕中**显示**（剥去外层括号，保留内容）。

| 项目 | 结果 |
|------|------|
| 字幕显示 | `点击专家->技能，搜索'3d模型查看'` |
| TTS 朗读 | `技能，搜索'3d模型查看'` |

### `[[text]]` 或 `〔〔text〕〕`

双层方括号内容**朗读**，但在字幕中**不显示**。与 `((text))` 对偶——用于给 TTS 补充发音词但不在屏幕显示。

```
[[点击]]技能
```

| 项目 | 结果 |
|------|------|
| 字幕显示 | `技能` |
| TTS 朗读 | `点击技能` |

> **用途**：Spark-TTS voice cloning 模式下，TTS 文本不宜过短（建议 ≥4 字），`[[...]]` 可给 TTS 补字，但不污染画面。

## 3. 换行 `\n`

在字幕显示时折行（ASS `\N`），TTS 朗读时替换为空格。

```
第一行\n第二行
```

| 项目 | 结果 |
|------|------|
| 字幕显示 | `第一行`（第一行）<br>`第二行`（第二行） |
| TTS 朗读 | `第一行 第二行` |

## 4. 同步点 `--N--`

单独一行。标记字幕和视频的同步点。

- **不录音**
- **不显示在字幕中**
- 将字幕条目分割，把下一行的字幕起始时间和当前的syncpoint时间同步。
- 必须与代码中的 `lib.syncpoint(page)` 一一对应

```
你可以告诉ai给模型更换材质
--1--
((提示词：))把汽车模型的外壳都换成黄金材质
于是外壳的材质就都换掉了
--2--
还可以让ai做更多
```

`--N--` 将字幕分割为 N+1 个 segment：
- segment 0: "你可以告诉ai给模型更换材质"（初始间隙 `INITIAL_GAP`）
- segment 1: "把汽车模型的外壳都换成黄金材质" + "于是外壳的材质就都换掉了"
- segment 2: "还可以让ai做更多"

每个 segment 内的字幕条目连续排列。`lib.syncpoint(page)` 记录视频时间戳，后续 burn 阶段将各 segment 对齐到对应 syncpoint 时间。

**计数校验**：`generate-subtitle.mjs` 会在生成时统计 `--N--` 数量和代码中 `syncpoint()` 调用次数，不匹配则报错退出。

## 5. 静音延时 `---x---`

单独一行。在上一句 TTS 结束后插入 `x` 毫秒的静音，然后才播下一句。

- **不录音**
- **不显示在字幕中**
- 替换默认的行间间隔（`INTER_LINE_GAP = 150ms`）
- 用于需要精确控制节奏的场景：模型加载、动画过渡等

```
展示了热门和收藏
---2000---
展示列表中的文件
```

上面示例中，"展示了热门和收藏" 说完后静音 2 秒，再播 "展示列表中的文件"。

## 优先级

当 `{voiceName}` 与括号语法同时出现时，`{voiceName}` 必须在**行首**最先解析：

```
{voiceName}((display only))actual text
```

解析顺序：`{voiceName}` 提取 → 括号语法处理显示/TTS 文本。

`--N--` 和 `---x---` 均为独立整行，不与其他语法混用。
